# Change Request Workflow

This is the one-by-one delivery model.

## Step 1 Request Intake
- Capture one change request at a time
- Define expected user behavior
- Define non-goals

## Step 2 Scope Lock
- Confirm acceptance criteria
- Confirm impacted pages/APIs/data
- Confirm test scenarios

## Step 3 Build
- Implement only current approved request
- Keep unrelated changes out of scope

## Step 4 Validate
- Run targeted tests for request
- Run core regression smoke checks

## Step 5 Approve
- Demo result
- Get explicit approval
- Only then move to next request
